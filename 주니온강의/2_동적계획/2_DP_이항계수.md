## 2-1 이항계수
### Quest
#### ❗ 용어 정리
##### 👉 동적 계획법 ?
* 메모이제이션
  * 이전에 만들어진거로 재활용해보자
  * 재귀 관계식(점화식)
* 상향식 방법
  * 작은 입력사례로 큰 것을 

* EX) 피보나치 수열, 이항계수

##### 👉 이항 계수?
* nCr
  * 조합의 성질
    * 서로 다른 n개에서 r개를 고르는 경우의수
    * 단, 순서는 고려하지 않는다
    ```cpp
    //1.
    C(n,r) == n! / r!(n-r)!
    //2.
    C(n, n) == C(n, 0) == 1
    //3.
    C(n, n-1) == C(n, 1) == n
    ```
  * 이항계수의 성질
    ```cpp
    //1. 
    for(int r = 0; r++ <= n; r++)
        res += c(n, r)
    >> res = 2ⁿ

    //2.
    C(n ,r) == C(n-1,r-1) + C(n-1,r)

    //3. 
    짝수행은 항의 개수가 짝수개이고, 
        좌우대칭이며 홀수행은 항의 개수가 홀수개이고, 
        가운데 숫자를 중심으로 좌우대칭이다.
    //4. 
    ```

#### ❗ 입력과 출력
* input 
    : n, r (단, n >= r)
* output 
    : C(n,r) 출력


#### ❗ 이항계수를 DP 기법으로 풀어보자
##### 👉 이항계수 점화식
* 이항계수의 점화식 (파스칼의 삼각형)
    ```
        C(n, r) n > r
            if(r <= 1) : 1
            else : C(n -1, r -1) + C(n - 1, r)
    ```

* **[이항계수 파이썬](./코드/이항계수.py)** 

##### 👉 좀더 효율적으로는 안될까?
1. 가운데 숫자를 중심으로 좌우대칭이다.
   * 시간복잡도 이득
2. An-1 번째 줄로도 An을 추론할 수 있지 않을까?
   * 공간 복잡도 이득
        ```cpp
        4 번째 : 1 4 6 4 1 이 한줄만 가지고 다음거는..
        5 번쨰 : 1 4 10 10 5 1 으로 추론 가능하지 않음?
        ```
